Accessibility

A11y

Semantic separator roles and forced-colors support.

Native Semantics <hr> has implicit role="separator". No extra ARIA needed for horizontal dividers.
Vertical Orientation Vertical dividers need aria-orientation="vertical" added explicitly for screen readers.
Decorative Dividers Purely visual dividers should use role="none" or aria-hidden="true" to hide from AT.
Forced Colors In Windows High Contrast mode, divider color maps to CanvasText for guaranteed visibility.

Use in Framework

Integration

Copy-paste ready code for React, Vue, HTML, and CSS token overrides.

<Divider variant="dashed" size="medium" />

{/* With label */}
<Divider variant="solid" size="base">
  <DividerLabel>OR</DividerLabel>
</Divider>

{/* Vertical */}
<Divider orientation="vertical" size="base" />
<DtfDivider variant="dashed" size="medium" />

<!-- With label -->
<DtfDivider variant="solid" size="base">
  <template #label>OR</template>
</DtfDivider>

<!-- Vertical -->
<DtfDivider orientation="vertical" size="base" />
<!-- Simple horizontal -->
<hr class="divider" data-variant="solid" data-size="base">

<!-- With label (use div, not hr) -->
<div class="divider" data-variant="solid" data-size="base" role="separator">
  <span class="divider__label">OR</span>
</div>

<!-- Vertical -->
<hr class="divider" data-variant="solid" data-size="base"
    data-orientation="vertical" aria-orientation="vertical">
/* Override divider tokens */
.my-app .divider {
  --div-color: var(--color-brand-200);
  --div-thickness: 2px;
  --div-indent-start: 24px;
  --div-label-color: var(--color-brand-500);
}
