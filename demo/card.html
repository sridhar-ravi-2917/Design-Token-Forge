Accessibility

A11y

Cards can be interactive (as links or buttons) with proper focus management.

Focus Ring Interactive cards (a.card, button.card, [data-interactive]) show a 2px focus ring with 2px offset on :focus-visible.
Forced Colors In Windows High Contrast, all variants get a visible ButtonText border. Ghost/flat variants get GrayText.
Semantic HTML Use <article> for standalone content cards. Use <a> for clickable cards. Use <section> for grouped content.
Disabled State data-disabled sets opacity: 0.5 and pointer-events: none. Add aria-disabled="true" for screen readers.

Use in Framework

Integration

Copy-paste ready code for React, Vue, HTML, and CSS token overrides.

<Card variant="raised" size="base">
  <CardHeader>Title</CardHeader>
  <CardBody>Content here</CardBody>
  <CardFooter>Actions</CardFooter>
</Card>
<DtfCard variant="raised" size="base">
  <DtfCardHeader>Title</DtfCardHeader>
  <DtfCardBody>Content here</DtfCardBody>
  <DtfCardFooter>Actions</DtfCardFooter>
</DtfCard>
<div class="card" data-variant="raised" data-size="base">
  <div class="card__header">Title</div>
  <div class="card__body">Content here</div>
  <div class="card__footer">Actions</div>
</div>
/* Override card tokens */
.my-app .card {
  --card-bg: var(--surface-bright-bg);
  --card-shadow-raised: var(--shadow-md);
  --card-radius-base: var(--radius-lg);
  --card-header-font-weight: var(--font-weight-bold);
}
